## 1. 这块做什么

用户配置 LLM 提供商的 API Key，Forgify 安全存储在本地。这是使用 AI 功能的前提。

---

## 2. 用户体验

### 2.1 入口

- 设置页 → "模型与 API" 标签
- 首次打开 Forgify 且没有配置任何 Key 时，对话页中间显示引导提示："先配置一个 API Key 才能开始对话 →"（点击跳转设置页）

### 2.2 提供商列表

页面展示支持的提供商卡片，每张卡片包含：
- 提供商 Logo + 名称
- 当前状态：未配置 / 已配置（显示 Key 的后 4 位）/ 连接异常
- "配置" / "编辑" 按钮

支持的提供商（第一版）：

| 提供商 | 需要填写 |
|---|---|
| Anthropic (Claude) | API Key |
| OpenAI | API Key |
| OpenAI 兼容端点 | API Key + Base URL |
| DeepSeek | API Key |
| Moonshot (Kimi) | API Key |
| Ollama（本地） | Base URL（无需 Key） |

### 2.3 配置流程

1. 点击"配置"→ 弹出配置抽屉（不跳转页面）
2. 填写 API Key（密码输入框，不明文显示）
3. 可选填写 Base URL（OpenAI 兼容端点需要）
4. 点击"测试连接"：
   - 成功：显示绿色✓ + "连接成功，可用模型：GPT-4o, GPT-4o-mini..."
   - 失败：显示红色✗ + 具体错误（如"401 Invalid API Key"）
5. 点击"保存"：Key 加密存储，抽屉关闭，卡片状态更新

### 2.4 Key 的安全展示

- 保存后的 Key 永远不明文显示，只显示 `sk-...****abcd`（前缀 + 后 4 位）
- 编辑时输入框为空，用户需重新输入完整 Key
- 删除 Key：卡片恢复"未配置"状态

### 2.5 连接异常

- 已配置的 Key 如果调用失败（过期、额度用完）：
  - 对话过程中出现错误时，错误消息提示"API Key 可能已失效，前往设置检查"
  - 设置页卡片状态显示"连接异常"（橙色）

## 4. 验收标准

- [ ] 可以配置至少一个提供商的 API Key
- [ ] 测试连接功能正常（成功和失败都有反馈）
- [ ] Key 保存后不明文显示
- [ ] 重启 Forgify 后 Key 仍存在（持久化）
- [ ] 删除 Key 后对应提供商恢复"未配置"状态
- [ ] 无 Key 时对话页有引导提示
