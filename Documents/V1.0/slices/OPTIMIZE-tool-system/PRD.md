# 工具系统优化 — 产品需求文档

**版本**：v1.0  
**日期**：2026-04-20  
**范围**：在 Tier 4 基础上补齐工具系统的完整体验

---

## 0. 当前问题

Tier 4 建立了工具的骨架（创建 → 执行 → 管理），但从产品角度看，三条核心链路有断裂：

1. **对话锻造链路断裂**：AI 生成代码 → 测试失败 → 用户必须手动复制错误让 AI 修复（应该一键修复）
2. **工具元数据不可编辑**：保存后无法改名、改描述、改分类（只有首次保存时可填）
3. **工具迭代无历史**：AI 多次生成同名函数会静默覆盖，用户看不到变化

---

## 1. 对话中的工具锻造增强

### 1.1 测试失败自动修复

测试失败后，结果区域增加 **"让 AI 修复"** 按钮：

```
┌─────────────────────────────────────────┐
│ ❌ TypeError: expected str, got int     │
│ 耗时：0.3s                              │
│                                          │
│ [🔧 让 AI 修复]   [复制错误]            │
└─────────────────────────────────────────┘
```

点击后，Forgify 自动将错误信息 + 当前代码作为用户消息发送给 AI：

```
我的工具 `send_email` 运行失败了：
TypeError: expected str, got int (第 12 行)

当前代码：
```python
def send_email(to: str, ...) -> dict:
    ...
```

请帮我修复。
```

AI 回复新代码 → forge 检测 → 自动更新草稿 → 用户可再次测试。**从 5+ 轮手动操作变为 1 次点击。**

### 1.2 代码迭代 Diff 展示

当 AI 为同一个函数生成新版本代码时，对话中显示变更摘要：

```
┌──────────────────────────────────────────┐
│ 🔄 已更新工具：send_email               │
│ 变更：+2 行 / -1 行                      │
│ · 新增 timeout 参数                       │
│ · 修改返回值格式                          │
│ [查看完整 diff]                           │
└──────────────────────────────────────────┘
```

### 1.3 AI 感知已有工具

系统提示中注入已有工具列表摘要，AI 回复前知道：
- 用户已有哪些工具
- 哪些工具是内置的
- 避免重复造轮子

注入格式（自动生成，用户不可见）：
```
[用户已有工具]
- send_email (邮件, 已测试)
- parse_excel (文件, 草稿)
- http_request (网络, 内置)
共 12 个工具。
```

## 2. 工具元数据管理

### 2.1 Inline 编辑（Notion 风格）

ToolMainView 的 header 区域：
- **名称**：点击变为输入框，blur 自动保存
- **描述**：点击变为输入框，支持多行
- **分类**：点击变为下拉选择器

不需要弹窗，不需要"编辑模式"。所见即所得。

### 2.2 标签系统

在固定分类之外，增加自由标签：
- 用户可以给工具打任意标签（如 "供应商"、"日报"、"Q2项目"）
- 标签可在工具库中作为筛选条件
- AI 创建工具时可自动建议标签

UI 位置：ToolMainView header 下方，以 tag chip 形式展示：
```
📦 发送报价邮件          ✅ 已测试
通过 SMTP 发送邮件       分类：邮件
[供应商] [Q2] [+ 添加标签]
```

### 2.3 对话控制元数据

用户在对话中可以用自然语言修改工具元数据：
- "把这个工具的名字改成 发送报价邮件"
- "描述改成：自动读取供应商报价Excel并汇总后发邮件"
- "分类改成邮件"
- "加一个标签：供应商"

AI 回复确认，后端直接更新。不需要代码块检测 — 这是纯元数据操作。

## 3. 参数文档化

### 3.1 参数描述

每个参数支持添加说明文字：

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| to | string | 是 | 收件人邮箱地址，支持多个用逗号分隔 |
| subject | string | 是 | 邮件主题 |
| timeout | int | 否 | 超时时间（秒），默认 30 |

说明来源：
1. AI 从 docstring 自动提取
2. 用户在参数 Tab 手动编辑
3. 对话中让 AI 补充

### 3.2 参数约束（可选）

高级用户可以为参数添加约束：
- `enum`：限定可选值（如 `method: GET|POST|PUT|DELETE`）
- `range`：数值范围（如 `timeout: 1-300`）
- `pattern`：正则（如 `email: ^\S+@\S+$`）

约束在测试时自动校验，不通过则提前报错。

## 4. 工具版本管理

### 4.1 自动版本记录

每次保存代码变更，自动创建版本快照：
- 版本号自增（v1, v2, v3...）
- 记录变更时间和变更摘要
- 可查看任意版本的代码

### 4.2 版本回滚

在代码 Tab 增加"历史版本"按钮：
```
[编辑] [历史版本 ▾]
```

点击展开版本列表：
```
v3  刚刚        当前版本
v2  2小时前     添加了 timeout 参数
v1  昨天        初始版本

[查看] [恢复到此版本]
```

## 5. 测试体验优化

### 5.1 保存测试参数

用户填过的测试参数可以保存为"测试用例"：
- 一个工具可以有多个测试用例（如 "正常情况"、"边界值"、"错误输入"）
- 下次打开测试 Tab，可以一键加载之前的用例

### 5.2 从对话中测试

ForgeCodeBlock 的测试结果卡片增加 **"重新测试"** 和 **"修改参数重测"** 按钮，用户不需要跳到资产页就能反复测试。

### 5.3 测试趋势

测试 Tab 底部显示简单的统计：
- 最近 20 次测试通过率
- 平均耗时
- 最后一次通过的时间

---

## 6. 验收标准

- [ ] 测试失败 → 点击"让 AI 修复" → AI 自动修复 → 新代码更新草稿
- [ ] AI 多次生成同名函数 → 对话中显示更新摘要（不是静默覆盖）
- [ ] ToolMainView 的名称/描述/分类可 inline 编辑
- [ ] 工具可以添加自由标签
- [ ] 代码保存自动创建版本，可查看历史，可回滚
- [ ] 对话中说"改名叫 xxx" → 工具名称更新
- [ ] 参数有文档说明（从 docstring 自动提取或手动填写）
- [ ] 测试参数可保存为用例
