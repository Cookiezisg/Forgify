# G4 · 错误修复对话 — 产品需求文档

**切片**：G4  
**状态**：待 Review

---

## 1. 背景

工作流运行失败时，用户应能通过对话让 AI 快速诊断问题并给出修复方案，而不是手动阅读日志、逐个节点排查。

---

## 2. 触发路径

两种入口都能进入错误修复对话：

1. **Canvas 顶部横幅** → 点击"通过对话修复"
2. **Inbox 中的失败通知** → 点击"打开对话修复"

---

## 3. AI 自动诊断

进入错误修复模式时，系统自动向对话注入当前错误信息（通过 `BuildContextInjection` 扩展），AI 在下一条回复中主动提供诊断：

```
[系统自动发出]：
  工作流"发送周报邮件"运行失败
  失败节点：gmail_send（工具节点）
  错误类型：node_failed
  错误信息：SMTP 认证失败，请检查 Gmail API Key

AI 回复：
  我看到你的工作流在"发送邮件"这一步失败了。错误是 SMTP 认证失败，
  通常原因是：
  1. Gmail API Key 已过期或权限不足
  2. 需要启用"不够安全的应用访问权限"

  建议你检查 B1 的 API Key 配置，或者我可以帮你修改这个节点改用 OAuth 方式...
```

---

## 4. AI 可执行的修复操作

AI 可以在对话中输出 `flow-definition` 代码块来直接修复工作流（复用 E2 机制），例如：
- 调整节点参数
- 替换失败的工具
- 修改重试次数
- 添加错误处理分支

---

## 5. 上下文注入内容

失败时注入的上下文：

```
[最近一次运行失败]
运行时间：2026-04-19 14:32
失败节点：node_3（工具: gmail_send）
错误类型：node_failed
错误信息：SMTP authentication failed
该节点输入：{"to":"boss@co.com","subject":"周报","body":"..."}
```

---

## 6. 验收测试

```
1. 工作流失败后点击"通过对话修复" → 对话打开并自动注入错误上下文
2. AI 第一条回复包含对失败原因的诊断和修复建议
3. AI 输出 flow-definition → 工作流自动更新（复用 E2 流程）
4. 修复后用户可直接从对话点击"再次运行"验证
```
