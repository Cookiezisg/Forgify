# F4 · 错误处理 — 产品需求文档

**切片**：F4  
**状态**：待 Review

---

## 1. 背景

工作流执行过程中可能遇到各种错误：工具执行失败、LLM 调用超时、网络错误等。良好的错误处理让用户能够快速定位问题，并通过对话修复工作流。

---

## 2. 错误分类

| 类型 | 示例 | 默认行为 |
|---|---|---|
| **节点执行错误** | 工具脚本抛出异常 | 停止整个工作流 |
| **LLM 调用错误** | API 超时、余额不足 | 停止并报告 |
| **参数错误** | `{{node_id.result}}` 为空 | 停止并报告 |
| **超时** | 节点超过配置的超时时间 | 停止并报告 |
| **Agent 迭代超限** | Agent 节点超过 max_iterations | 停止该节点，继续或停止工作流（可配置） |

---

## 3. 错误展示

### 画布上

- 失败节点：红色边框
- 点击失败节点 → 配置面板切换为"错误详情"视图：
  - 错误类型
  - 错误信息（完整 stack trace，可复制）
  - 发生时间
  - 该节点的输入参数（帮助排查）

### 顶部横幅

工作流失败时顶部出现横幅：

```
❌ 工作流运行失败 · 节点"发送邮件"出错 · [查看详情]  [通过对话修复]
```

---

## 4. 重试配置

每个节点可配置重试策略（在节点配置面板中）：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 最大重试次数 | 0（不重试）| 1-5 次 |
| 重试间隔 | 5 秒 | 固定间隔 |
| 重试触发条件 | 所有错误 | 可选"仅网络错误" |

---

## 5. "通过对话修复" 入口

顶部横幅的"通过对话修复"按钮：
- 打开（或切换到）该工作流绑定的对话
- AI 自动感知运行错误（通过状态注入）
- AI 提示：哪个节点出错、错误原因、建议修复方案

此功能与 G4（错误修复对话）联动，G4 专门负责 AI 诊断逻辑。

---

## 6. 验收测试

```
1. 工具节点执行失败 → 节点变红 → 顶部横幅出现
2. 点击失败节点 → 面板显示错误类型、错误信息、输入参数
3. 节点配置重试 2 次 → 失败时自动重试 2 次后才标记为失败
4. Agent 超过 max_iterations → 节点失败，错误信息"超过最大迭代次数"
5. 点击"通过对话修复" → 切换到对话，AI 给出修复建议
```
