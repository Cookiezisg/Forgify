# 对话-工具分屏视图 — 产品需求文档

**版本**：v1.0  
**日期**：2026-04-20  
**参考**：PRD_1.0.md 第 3.3 节、第 4.2 节

---

## 0. 背景

PRD 核心哲学："**对话是主角，画布是镜子。**" 

当前问题：用户在对话中让 AI 锻造工具后，代码只出现在对话气泡里，随着后续消息被推走。要看代码或测试必须去资产页。**锻造过程被割裂成了"聊天"和"管理"两个分离的场景。**

目标：让绑定了工具的对话变成 **ChatGPT Canvas 模式** — 左边持续聊天，右边工具代码实时跟随。用户边聊边看代码变化，边测试边迭代，不需要离开对话。

---

## 1. 整体布局

### 1.1 未绑定工具的对话（不变）

```
┌────────────────────────────────────────────┐
│                对话区域                      │
│                                            │
│  用户消息...                                │
│  AI 回复...                                │
│                                            │
│  [输入框]                                  │
└────────────────────────────────────────────┘
```

和现在完全一样，没有右侧面板。

### 1.2 绑定了工具的对话 — 展开状态

```
┌───────────────────────┬─┬──────────────────┐
│      对话区域          │ │   工具面板        │
│                       │拖│                  │
│  用户：帮我做个发邮件  │拽│  📦 发送邮件     │
│  AI：生成了代码 →     │线│  ✅ 已测试       │
│                       │ │  ─────────────── │
│  用户：加个超时参数    │ │  [代码] [参数] [测试] │
│  AI：已更新 →         │ │                  │
│                       │ │  def send_email( │
│  用户：测试一下        │ │    to: str,      │
│  [测试结果卡片]        │ │    subject: str, │
│                       │ │    timeout: int   │
│  [输入框]              │ │  ) -> dict:      │
│                       │ │    ...           │
└───────────────────────┴─┴──────────────────┘
```

- **左侧**：完整对话界面（消息列表 + 输入框 + 压缩提示）
- **中间**：可拖拽的分割线，用户可以调节左右宽度比
- **右侧**：完整的工具视图（复用现有 ToolMainView），包含代码 Tab、参数 Tab、测试 Tab

### 1.3 绑定了工具的对话 — 收起状态

```
┌──────────────────────────────────────┬────┐
│            对话区域                    │ 📦 │
│                                      │ 发 │
│  （完整宽度的对话界面）                │ 送 │
│                                      │ 邮 │
│                                      │ 件 │
│  [输入框]                             │    │
└──────────────────────────────────────┴────┘
                                        36px
```

- 右侧变为 **36px 宽的竖条**
- 竖条上显示工具图标 📦 和工具名（竖排文字）
- **点击竖条**即可展开回完整工具面板
- 对话区域获得完整宽度

---

## 2. 触发条件

| 条件 | 是否显示工具面板 |
|---|---|
| 对话没有绑定任何资产 | ❌ 不显示 |
| 对话绑定了工具（`assetType='tool'`） | ✅ 显示 |
| 对话绑定了工作流（`assetType='workflow'`） | ⏳ 预留，当前显示占位提示 |
| AI 在对话中创建了工具（自动绑定后） | ✅ 面板自动出现 |
| 用户从资产页"新建对话"（预绑定） | ✅ 面板从一开始就在 |

---

## 3. 工具面板内容

### 3.1 面板头部

```
┌──────────────────────────────────┐
│ 📦 发送邮件        ✅ 已测试  [◀] │
│ 通过 SMTP 发送邮件  分类：邮件    │
└──────────────────────────────────┘
```

- 工具名称 + 状态徽章（草稿/已测试/失败）
- 描述 + 分类
- **[◀] 收起按钮**：点击将面板收起为竖条

### 3.2 三个 Tab（复用 ToolMainView）

**代码 Tab：**
- Monaco Editor，语法高亮
- 非内置工具可编辑（有编辑/保存/取消按钮）
- AI 每次更新工具代码后，编辑器内容实时刷新

**参数 Tab：**
- 参数列表表格（名称/类型/必填/说明）

**测试 Tab：**
- 参数填写表单
- ▶ 运行测试按钮
- 测试结果展示
- 最近测试记录

### 3.3 实时同步

AI 在对话中生成新代码 → 后端 `forge.code_detected` 事件 → 前端收到事件后：
1. 对话气泡中的代码块正常显示（作为对话记录）
2. **右侧工具面板自动刷新**，显示最新代码
3. 面板状态回到代码 Tab（如果不在的话）

用户不需要手动操作就能看到代码更新。

## 4. 交互细节

### 4.1 宽度调节

- 默认比例：对话区占 50%，工具面板占 50%
- 用户可拖拽中间分割线调整
- 对话区最小宽度：300px
- 工具面板最小宽度：320px（Monaco Editor 最小可用宽度）
- 拖拽到最右边 → 自动收起为竖条

### 4.2 收起/展开

- 点击面板头部的 **[◀]** 按钮：收起为竖条
- 点击竖条的任意位置：展开为完整面板
- 收起/展开状态**按对话记忆**（存 localStorage，key 为 `forgify.toolPanel.${conversationId}`）
- 切换到其他对话再切回来，状态保持

### 4.3 绑定变化时的行为

| 事件 | 行为 |
|---|---|
| 对话中 AI 创建了工具（首次绑定） | 面板从右侧**滑入**出现（展开状态） |
| 用户在绑定横幅点"解除关联" | 面板**滑出**消失 |
| 切换到未绑定的对话 | 无面板 |
| 切换到绑定了工具的对话 | 面板出现（按记忆的收起/展开状态） |

### 4.4 对话气泡中的代码块

有右侧面板时，对话气泡中的 Python 代码块：
- **仍然显示**（作为 AI 回复内容的一部分）
- **不再显示 "测试运行" / "保存为工具" 按钮**（这些操作已在右侧面板）
- 这避免了按钮重复

没有右侧面板时（未绑定对话），代码块按钮保持现有行为。

---

## 5. 和绑定横幅的关系

当前对话顶部有绑定横幅 `📎 已关联到：xxx [解除]`。

**有工具面板时：** 绑定横幅**隐藏**，因为面板头部已经显示了工具信息 + 面板本身就是绑定的视觉体现。解除关联的操作移到面板头部的 `...` 菜单中。

**没有工具面板时（如绑定了工作流）：** 绑定横幅正常显示。

---

## 6. 边缘情况

| 情况 | 处理 |
|---|---|
| 绑定的工具被删除 | 面板显示"工具已删除"提示，可选关闭面板 |
| 内置工具 | 面板中代码 Tab 只读，无编辑按钮 |
| 窗口很窄（< 620px） | 自动收起为竖条 |
| 对话绑定了工作流 | 面板显示占位："工作流画布（开发中）" |

---

## 7. 验收标准

- [ ] 未绑定对话：无右侧面板，和现在一样
- [ ] 绑定工具的对话：左对话 + 右工具面板分屏显示
- [ ] 拖拽分割线可调节左右宽度
- [ ] 点击 [◀] 收起为 36px 竖条，点竖条展开
- [ ] 收起/展开状态按对话记忆，切换对话后保持
- [ ] AI 生成新代码 → 右侧面板代码自动刷新
- [ ] 有面板时，对话气泡代码块不显示测试/保存按钮
- [ ] 解除绑定 → 面板消失，对话恢复全宽
- [ ] AI 创建工具触发绑定 → 面板自动滑入
