# F2 · 手动运行 — 产品需求文档

**切片**：F2  
**状态**：待 Review

---

## 1. 背景

工作流编写完成后，用户需要能够手动触发一次运行来验证效果。手动运行是工作流执行引擎的最基础场景，也是后续定时/事件触发的运行时基础。

---

## 2. 用户操作流程

1. 在工作流 Canvas 顶部工具栏点击 **"运行一次"** 按钮
2. 如果工作流有手动触发节点且需要输入参数，弹出"运行参数"对话框
3. 确认后工作流开始执行
4. 画布上各节点实时显示状态（运行中 / 成功 / 失败）
5. 每个节点执行完成后，可点击查看该节点的输入/输出
6. 整个工作流完成后，顶部显示"运行成功"或"运行失败 + 错误信息"

---

## 3. 节点状态实时反馈

执行过程中，画布节点通过颜色和动画反映状态：

| 状态 | 视觉 |
|---|---|
| 等待中 | 默认灰色边框 |
| 运行中 | 蓝色边框 + 脉冲动画 |
| 成功 | 绿色边框 |
| 失败 | 红色边框 |
| 跳过（条件不满足） | 灰色边框 + 斜线纹理 |

---

## 4. 节点输入/输出查看

执行后点击任意节点，配置面板切换为"运行结果"视图，显示：
- **输入**：该节点实际收到的参数（变量已解析）
- **输出**：该节点返回的 result
- **耗时**：执行时长（毫秒）
- **错误信息**（如失败）

---

## 5. 运行参数输入

`trigger_manual` 节点可配置"输入参数 schema"，运行时弹出对话框让用户填写：

```
运行参数
─────────────────
  recipient_email  [                    ]
  subject          [                    ]

              [取消]  [开始运行]
```

无参数配置时直接开始运行，不弹对话框。

---

## 6. 限制

- 同一工作流同时只允许一次手动运行
- 运行中再次点击"运行一次"：提示"运行中，请等待完成"
- 运行超时：单节点默认 30 秒，整体工作流默认 10 分钟

---

## 7. 验收测试

```
1. 点击"运行一次" → 工作流开始执行 → 节点依次变蓝→绿
2. 条件节点路由到 "yes" 分支 → "no" 分支节点状态显示"跳过"
3. 工具节点执行失败 → 节点变红 → 顶部显示错误
4. 点击已完成节点 → 查看该节点的输入/输出
5. trigger_manual 配置了参数 schema → 运行前弹出参数填写对话框
6. 运行中再次点击"运行一次" → 提示"运行中"，不重复执行
```
