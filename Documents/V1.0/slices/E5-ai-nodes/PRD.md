# E5 · AI 节点类型 — 产品需求文档

**切片**：E5  
**状态**：待 Review  
**依赖**：E3、B2、B3  
**下游**：F1

---

## 1. LLM 节点

一次性调用大模型，用于文本生成、总结、分类、判断。

配置：
- System prompt（支持引用上游数据：`{{node_2.result}}`）
- 模型选择（默认使用工作流 LLM 节点模型）
- 输出格式：纯文本 / JSON（指定 schema）
- 温度（temperature）

示例用途：
- 总结多封邮件的要点
- 判断报价是否合理（返回 true/false）
- 生成格式化报告

---

## 2. Agent 节点

给 LLM 一个工具子集和一个目标，由 LLM 自主决定如何使用这些工具来完成目标。适合路径不确定的子任务。

配置：
- 目标描述（`从邮件列表中找出所有含报价的邮件，提取供应商名称和总金额`）
- 可用工具集（从工具库选择 1-10 个）
- 最大迭代次数（防止无限循环，默认 10）
- 策略：ReAct（思考-行动-观察循环）

运行时行为：
- 节点运行时画布上该节点显示"思考中..."动画
- Agent 的中间步骤（调用了哪些工具、观察到什么）记录到运行历史，但不显示在对话中
- 最终输出为 Agent 认为任务完成时返回的结果

**与 LLM 节点的区别**：LLM 节点一次调用固定返回；Agent 节点可以多轮调用工具，直到它认为目标完成。

---

## 3. 验收标准

- [ ] LLM 节点：传入文本 → AI 总结后输出，输出可被下游节点引用
- [ ] LLM 节点 JSON 输出模式：返回符合指定 schema 的 JSON
- [ ] Agent 节点：给定工具 [http_request, json_parse] 和目标 → Agent 自动完成
- [ ] Agent 节点：超过最大迭代次数时返回错误，不无限运行
