# K1 · 模型设置 — 产品需求文档

**切片**：K1  
**状态**：待 Review

---

## 1. 背景

Forgify 使用 Eino 作为 LLM 编排层，支持多个模型提供商。用户需要能配置使用哪个模型、填写 API Key，并可以设置不同场景下的模型偏好。

---

## 2. 设置入口

侧边栏底部的"⚙️ 设置"图标，点击打开设置页面，分"模型"和"通用"两个标签页。

---

## 3. 支持的模型提供商

| 提供商 | 模型 | 需要 API Key |
|---|---|---|
| Anthropic | claude-3-5-sonnet, claude-3-opus | 是 |
| OpenAI | gpt-4o, gpt-4-turbo | 是 |
| 本地（Ollama） | 自定义（用户填写模型名）| 否 |

---

## 4. 配置项

| 配置项 | 说明 |
|---|---|
| **对话模型** | 用于普通对话和工作流 AI 创建的模型 |
| **LLM 节点模型** | 工作流中 LLM 节点使用的模型（可与对话模型不同）|
| **Agent 节点模型** | Agent 节点专用（推荐能力强的模型）|
| **Ollama 地址** | 本地 Ollama 服务地址（默认 http://localhost:11434）|

每个模型选择器显示提供商 + 模型名，并有连接测试按钮。

---

## 5. 连接测试

点击"测试连接"：
- 发送一条简短测试消息（"ping"）
- 成功：绿色"连接正常 ✓"+ 响应延迟
- 失败：红色"连接失败"+ 错误原因（如"API Key 无效"）

---

## 6. 验收测试

```
1. 切换模型提供商 → API Key 输入框出现/消失
2. 输入合法 API Key → 测试连接成功 → 绿色提示
3. 输入错误 API Key → 测试连接失败 → 红色提示
4. 选择 Ollama → 不需要 API Key，显示地址输入框
5. 保存设置后，对话使用新模型
6. LLM 节点设置与对话设置独立
```
