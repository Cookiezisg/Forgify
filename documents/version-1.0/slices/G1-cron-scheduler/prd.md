# G1 · Cron 定时调度 — 产品需求文档

**切片**：G1  
**状态**：待 Review

---

## 1. 背景

工作流可以配置定时触发（`trigger_schedule` 节点），Cron 调度器负责在正确的时间点启动对应的工作流运行。

---

## 2. 调度器行为

- 应用启动时，调度器自动扫描所有状态为 `deployed` 且含 `trigger_schedule` 节点的工作流
- 按节点配置的 Cron 表达式注册定时任务
- 到达触发时间时，自动调用执行引擎运行该工作流
- 工作流部署（状态从 `draft` → `deployed`）时，动态注册新任务
- 工作流暂停/归档时，动态取消任务

---

## 3. Cron 表达式格式

支持标准 5 字段 Cron（分 时 日 月 周）：

```
0 9 * * *       每天 09:00
0 * * * *       每小时
0 9 * * 1       每周一 09:00
*/5 * * * *     每 5 分钟
```

最小精度：**1 分钟**（不支持秒级）。

---

## 4. 时区

调度器使用**本机时区**，不做时区转换。

---

## 5. 错过执行处理

如果应用在本应触发的时间段内没有运行（关机），**不会补偿执行**，直接等待下一次触发时间。

---

## 6. 验收测试

```
1. 部署含 trigger_schedule（cron: "* * * * *"）的工作流 → 每分钟自动运行
2. 暂停工作流 → 定时任务停止
3. 重新部署 → 定时任务重新注册
4. 应用关机期间错过的触发 → 重启后不补偿，等待下次触发
5. 无效 cron 表达式 → 部署失败，提示错误
```
