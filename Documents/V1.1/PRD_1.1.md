# Forgify V1.1 — 产品迭代需求文档

**版本**：v1.1  
**日期**：2026-04-20  
**基于**：PRD_1.0.md (v0.3) + Tier 1-4 实现经验  
**性质**：架构级迭代，非增量补丁

---

## 0. 迭代动机

V1.0 的 Tier 1-4 验证了核心能力（对话、工具锻造、Python 沙箱），但暴露了三个根本问题：

1. **页面模型过于静态** — 一个 Tab 对应一个固定页面，无法表达"对话→工具→工作流"的层层关联。用户被迫在侧边栏 Tab 间反复切换，上下文断裂。

2. **锻造过程割裂** — 用户在对话中让 AI 生成代码，但代码随消息滚走。测试和编辑必须去资产页。"边聊边造"的核心体验不存在。

3. **工具管理粗糙** — 保存后不能改名、没有版本、AI 不知道已有工具、测试失败需手动复制错误给 AI。

**V1.1 的目标：用一次架构级重构，同时解决这三个问题。**

## 1. 新前端架构：Tab + 固定布局

### 1.1 设计哲学

借鉴浏览器的思想：**内容区是多标签页，每个 Tab 是一个独立的工作空间。**

用户可以同时打开多个 Tab，自由切换，每个 Tab 有固定的布局类型。不会出现"挤来挤去"或不确定的面板组合。

### 1.2 整体结构

```
┌──────┬─────────────────────────────────────────────┐
│      │ [Tab1: 报价讨论] [Tab2: send_email] [Tab3:+] │
│ Nav  ├─────────────────────────────────────────────┤
│ Bar  │                                             │
│      │              Tab 内容区                      │
│      │         （由布局类型决定）                    │
│      │                                             │
└──────┴─────────────────────────────────────────────┘
```

- **Nav Bar**（左侧，不变）：Home / 对话 / 资产 / Inbox / 设置
- **Tab Bar**（内容区顶部，新增）：浏览器风格的标签栏
- **Tab 内容区**：根据布局类型渲染

### 1.3 五种布局类型

每个 Tab 的布局在创建时确定，**不会动态变化**（除了绑定触发的升级）：

| 布局 | 结构 | 触发方式 |
|---|---|---|
| **Chat** | 全宽对话 | 新建对话、点击对话列表 |
| **Tool** | 全宽工具视图 | 从工具库打开工具 |
| **Workflow** | 全宽工作流画布 | 从工作流列表打开 |
| **Chat+Tool** | 左对话 右工具 | 对话绑定了工具（自动或手动） |
| **Chat+Workflow** | 左对话 右画布 | 对话绑定了工作流 |

**布局升级规则：**
- Chat 布局 → 对话中 AI 创建了工具 → 自动升级为 Chat+Tool
- Chat 布局 → 对话中 AI 创建了工作流 → 自动升级为 Chat+Workflow
- 用户解除绑定 → Chat+Tool 降级回 Chat

### 1.4 Tab 管理规则

**打开新 Tab：**
- Nav 侧边栏点"对话" → 显示对话列表，点击某对话 → 打开 Chat Tab
- Nav 侧边栏点"资产" → 显示工具/工作流列表，点击某工具 → 打开 Tool Tab
- 对话中 AI 创建工具 → 当前 Chat Tab 升级为 Chat+Tool（不开新 Tab）
- 工具页点"新建对话" → 开新 Chat+Tool Tab（预绑定）
- 工作流画布点某节点的工具 → 开新 Tool Tab

**Tab 数量：** 无硬限制，但 Tab Bar 超出时水平滚动（同浏览器）

**关闭 Tab：** 每个 Tab 有 × 关闭按钮。最后一个 Tab 关闭后显示空状态（"打开一个对话或资产"）

**Tab 状态保持：** 切换 Tab 时，每个 Tab 的滚动位置、输入框内容、编辑状态全部保留（不 unmount）

### 1.5 Nav 侧边栏的新角色

Nav 从"切换页面"变为"**打开内容列表 + 创建新 Tab 的入口**"：

```
┌──────┐
│ 🏠   │  Home → 开 Home Tab（固定 Tab，不可关闭）
│      │
│ 💬   │  对话 → 侧边栏展开对话列表，点击开 Chat Tab
│      │
│ 📦   │  资产 → 侧边栏展开工具/工作流列表，点击开 Tool/Workflow Tab
│      │
│ 📥   │  Inbox → 开 Inbox Tab（固定 Tab）
│      │
│ ⚙️   │  设置 → 开 Settings Tab（固定 Tab）
└──────┘
```

点击 Nav 图标 → 侧边栏显示该分类的列表 → 点击列表项 → 在 Tab Bar 打开对应 Tab

### 1.6 分屏布局细节（Chat+Tool 和 Chat+Workflow）

```
┌─────────────────────┬─┬───────────────────┐
│     对话区域         │↔│    工具/画布区域    │
│                     │ │                   │
│  消息列表            │ │  代码编辑器 /      │
│  输入框              │ │  工作流画布        │
│                     │ │                   │
└─────────────────────┴─┴───────────────────┘
          50%          ↔        50%
```

- 中间分割线可拖拽调节比例
- 左侧最小 300px，右侧最小 320px
- 右侧可收起为 36px 竖条（点击展开）
- 收起状态竖条显示工具/工作流图标 + 名称

## 2. 工具锻造闭环

### 2.1 AI 感知已有工具

每次对话调用 LLM 前，系统提示中注入用户工具库摘要：

```
[用户已有工具]
- send_email (邮件, 已测试)
- parse_excel (文件, 草稿)
- http_request (网络, 内置)
共 12 个工具。如用户需要的功能已有工具可用，优先推荐使用。
```

这让 AI 能说"你已经有一个 send_email 工具，要不要直接用？"而不是重复造一个。

### 2.2 测试失败一键修复

Chat+Tool 布局中，右侧测试 Tab 测试失败后：

```
┌─────────────────────────────────────────┐
│ ❌ TypeError: expected str, got int     │
│ 耗时：0.3s                              │
│                                          │
│ [🔧 让 AI 修复]   [复制错误]            │
└─────────────────────────────────────────┘
```

点击"让 AI 修复"：
1. 自动在左侧对话中发送消息："工具 `send_email` 运行失败：{错误详情}，请修复。"
2. AI 回复新代码
3. 右侧代码面板自动刷新为最新版本
4. 用户可立即再次测试

**整个修复循环在同一个 Tab 内完成，不需要切换页面。**

### 2.3 代码迭代可见

AI 为同一函数生成新版本时，对话中显示更新卡片（而非静默覆盖）：

```
┌──────────────────────────────────────────┐
│ 🔄 已更新工具：send_email               │
│ 变更：+2 行 / -1 行                      │
│ · 新增 timeout 参数                       │
│ · 修改返回值格式                          │
└──────────────────────────────────────────┘
```

右侧代码面板同步更新，用户始终看到最新代码。

### 2.4 对话气泡中代码块的变化

**Chat+Tool 布局时：**
- 对话中的 Python 代码块仍然显示（作为对话记录）
- **不再显示"测试运行"/"保存为工具"按钮**（这些操作在右侧面板）

**普通 Chat 布局时（未绑定工具）：**
- 代码块保留"测试运行"/"保存为工具"按钮（现有行为不变）

## 3. 工具元数据增强

### 3.1 Inline 编辑

工具视图（无论是独立 Tool Tab 还是 Chat+Tool 右侧面板）的 header 区域：

- **名称**：点击变输入框，blur 自动保存（Notion 风格）
- **描述**：点击变输入框，支持多行
- **分类**：点击变下拉选择器

所见即所得，不需要弹窗，不需要"编辑模式"按钮。

### 3.2 标签系统

在固定分类之外，增加自由标签：

```
📦 发送报价邮件          ✅ 已测试
通过 SMTP 发送邮件       分类：邮件
[供应商] [Q2项目] [+ 添加标签]
```

- 用户可以打任意标签
- 工具库可按标签筛选
- AI 创建工具时可自动建议标签

### 3.3 版本管理

每次代码保存自动创建版本快照：

- 版本号自增（v1, v2, v3...）
- 记录变更摘要（"+2 行 / -1 行"）
- 代码 Tab 增加"历史版本"入口，可查看任意版本、可回滚

### 3.4 参数文档化

参数 Tab 中每个参数支持添加描述：

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| to | string | 是 | 收件人邮箱，多个用逗号分隔 |
| timeout | int | 否 | 超时秒数，默认 30 |

说明来源：AI 从 docstring 自动提取 + 用户手动编辑。

### 3.5 测试用例

测试 Tab 支持保存参数组合为"测试用例"：
- 一个工具多个用例（如"正常情况"、"边界值"）
- 下次测试可一键加载

## 4. 实现优先级

### Phase 1：前端架构重构（Tab 系统）

这是基础设施，后续所有功能都依赖它。

| # | 内容 |
|---|---|
| 1 | Tab 状态管理（TabContext：打开/关闭/切换/布局类型） |
| 2 | Tab Bar 组件（标签栏 UI，关闭按钮，+ 新建） |
| 3 | Nav 侧边栏改造（从"切换页面"变为"打开列表+创建 Tab"） |
| 4 | 5 种 Layout 组件（Chat / Tool / Workflow / Chat+Tool / Chat+Workflow） |
| 5 | Tab 状态保持（切换时不 unmount，保留滚动和输入状态） |

### Phase 2：Chat+Tool 分屏

依赖 Phase 1 的 Tab 系统。

| # | 内容 |
|---|---|
| 6 | 分屏容器（可拖拽分割线 + 收起为竖条） |
| 7 | 绑定触发自动升级（Chat → Chat+Tool） |
| 8 | AI 修复按钮（测试失败 → 自动发消息给 AI） |
| 9 | 代码更新实时同步（forge 事件 → 右侧面板刷新） |

### Phase 3：工具系统增强

可以和 Phase 2 并行。

| # | 内容 |
|---|---|
| 10 | Inline 编辑名称/描述/分类 |
| 11 | 标签系统（DB + UI） |
| 12 | 版本管理（自动快照 + 回滚） |
| 13 | AI 感知已有工具列表 |
| 14 | 测试用例保存/加载 |

### Phase 4：对话中的代码块行为调整

依赖 Phase 1 + Phase 2。

| # | 内容 |
|---|---|
| 15 | Chat+Tool 布局下隐藏气泡中的测试/保存按钮 |
| 16 | 代码更新卡片（非静默覆盖） |
| 17 | 参数文档编辑 |

## 5. 对 V1.0 的影响

### 5.1 替代的组件

| V1.0 组件 | V1.1 变化 |
|---|---|
| `SidebarNav`（Tab 切换器） | 改为列表展开入口 |
| `App.tsx` 的 LeftPanel/MainContent | 替换为 TabBar + TabContent |
| `ChatPage.tsx` 的 ChatContent | 拆为独立 Layout 组件 |
| `AssetsPage.tsx` | 拆为 ToolPage + WorkflowPage |
| `BindingIndicator` | 被分屏面板头部替代 |
| `ForgeCodeBlock` 的测试/保存按钮 | 在 Chat+Tool 布局下隐藏 |

### 5.2 保留不变的

| 组件 | 原因 |
|---|---|
| ToolMainView | 直接复用为右侧面板内容和独立 Tool Tab |
| useChat hook | 对话逻辑不变 |
| ChatInput / MessageList / MessageItem | 聊天组件不变 |
| 所有后端 API | 完全不变 |
| ChatContext | 保留，Tab 系统在其上层 |

### 5.3 新增的核心概念

| 概念 | 说明 |
|---|---|
| `TabContext` | 管理所有打开的 Tab（ID、类型、布局、参数） |
| `TabBar` | 标签栏 UI 组件 |
| `Layout` | 5 种布局的容器组件 |
| `SplitLayout` | 可拖拽分屏 + 可收起的布局容器 |

## 7. 验收标准

### Tab 系统
- [ ] 可以同时打开多个 Tab，Tab Bar 显示所有标签
- [ ] 点击 Nav 侧边栏 → 展开对应列表 → 点击项目 → 打开 Tab
- [ ] 关闭 Tab → 标签消失，切换到相邻 Tab
- [ ] 切换 Tab → 内容状态保持（滚动位置、输入内容）
- [ ] 固定 Tab（Home、Inbox、Settings）不可关闭

### 分屏布局
- [ ] 对话绑定工具 → Chat Tab 自动升级为 Chat+Tool
- [ ] 左右可拖拽调节宽度
- [ ] 右侧可收起为 36px 竖条，点击展开
- [ ] AI 生成新代码 → 右侧面板代码自动刷新
- [ ] Chat+Tool 下对话气泡不显示测试/保存按钮

### 工具增强
- [ ] 工具名称/描述/分类可 inline 编辑
- [ ] 可添加和删除标签
- [ ] 代码保存自动创建版本快照
- [ ] 测试失败 → 点"让 AI 修复" → AI 自动在对话中修复
- [ ] AI 知道用户已有哪些工具
